Project Overview

The Challenge

A Tech Support company was struggling with their existing paper and spreadsheet appointment system, which led to data inconsistencies, scheduling conflicts, and difficulty in managing client information efficiently. They needed a modern, reliable database solution to streamline their operations.

The Solution

I developed a comprehensive relational database system using Microsoft Access with two main interconnected tables: TblClients and TblAppointments. The solution featured robust validation rules, user-friendly forms, efficient queries, and detailed reports to completely transform their appointment management process.

Key Features Implemented

  • Client Management with comprehensive validation
  • Appointment Scheduling with time restrictions
  • One-to-Many relationship enforcement
  • User-friendly navigation forms
  • Automated data validation and error handling
  • Search and filter capabilities

Database Architecture

Implemented a normalized two-table structure with TblClients (25 records) and TblAppointments (25 records). Established one-to-many relationship using ClientID as primary key in clients table and foreign key in appointments table.

  • Primary/Foreign key relationships
  • Referential integrity enforcement
  • Proper data type selection
‚úÖ

Advanced Validation Rules

Created comprehensive validation rules to ensure data accuracy: name fields accept only alphabetic characters, appointment times restricted to 8AM-6PM, email format validation, and age verification (18+).

  • Appointment time: Between #08:00:00# And #18:00:00#
  • Date validation: >=Date() (no past dates)
  • Email format: Like "*?@?*.com"
  • Age verification: <=DateAdd("yyyy",-18,Date())

üéØ What I Learned

This project significantly enhanced my understanding of relational database design principles, particularly the importance of proper table relationships and data validation. I gained valuable experience in creating user-centered interfaces and implementing comprehensive testing procedures to ensure system reliability.

üöÄ Challenges Overcome

The most significant challenge was implementing robust validation rules while maintaining user-friendliness. I had to balance strict data integrity requirements with intuitive user experience, ensuring that error messages were helpful rather than frustrating for staff members.

üí° Key Insights
